1. Remove all service local try/catch
    - 11000 that cannot be handled generically
    - Catches for non-fatal side effects (WhatsApp dispatch, email notifications)
    - Compensating delete/rollback when a multi-step operation partially succeeds

try {
  await Model.create(data);
} catch (err) {
  if ((err as { code?: number }).code === 11000) {
    throw new AuthContextError('domain.errors.duplicate', 409);
  }
  throw err;
}

    requireAuthContext / requireRole
        ↓

        ↓
    connectToMongoDB

        ↓

        ↓

    - still call requireAuthContext on every service
        - Defense in depth
        - Self-documenting contract

3. Auth Guard Style
    - requireAuthContext(event) for every protected service
    - requireRole(event, roles) for service requires a specific role regardless of ownership
    - getAuthContext for non-jwt-protected service that response (include specific fields or not) differently 
    - Bespoke JWT parsing (getBearerToken + jwt.verify directly) for auth verify challenge 
    - local auth wrapper for compound check that repeats across multiple handlers in the same domain
